Zed vs Emacs

最近 Zed 成了编辑器的“新贵”，Emacs China 论坛也在讨论 Zed 与 Emacs 的优劣，我也发表了一些自己的意见，摘录在这里（稍有润色）：

需求都是不定且在不停增长的。有段时间我也是想着“追新”，但发现这与夸父追日无异。相对于这些“新贵”，我现在更愿意优先使用成熟稳定且灵活的软件，比如 emacs、vim 和 linux。最近流行用 rust 造各种轮子，自己都鲜有尝试。它们创新并不多，对工作而已，收益甚微（当然不失为一种学习方法）。

我几乎不是很有意愿去尝试 zed，原因大概如下：

  1. Zed 固然没有历史负担，但如果其要发展，往后必然也会变的臃肿，即时将如何面对，再换个“新贵”？
  2. “新”也就意味着不稳定和功能不全，所以只能你去就它，而不是它来就你。
  3. 在我看来 zed 唯二比 emacs 有优势的地方是 “快” 和 “开箱即用”。在大部分任务上，emacs 够“快”了。我已经能顺手的使用 emacs，也就没有开箱即用的问题。

如果嫌弃 emacs 臃肿，可以自己裁剪个满足自身需求的轻量发行版。这就是它来就我，也就是自由软件中“自由”的其中一层意思。

我喜欢 emacs 和 vim 的另一个强烈的理由是，它们自有一种“编辑”的理念。在使用它们的过程中，可以深入的理解什么是编辑以及如何编辑，得到一种编辑思维的提升。这是很多其他主流编辑器不具有的。

不过我个人希望大家多去造轮子，尝试使用新的软件。这样才能发展出新的东西来。自己有更重要（对自己而已）事去做，没了是“试新”的动力和时间。

我相信很多人也会有我这般看法，但并不意味着其他看法是错的，只是不同情景下做出的不同选择而已。
